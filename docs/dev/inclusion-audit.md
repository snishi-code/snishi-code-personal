# simple-ledger — Inclusion audit

UI 変更時に確認する、色彩・色覚多様性・宗教/文化配慮の観点。正本は実装ではなく
`simple-ledger-src/src/ui/tokens.css` と各 UI 仕様だが、このファイルをチェックリストとして使う。

## 色彩・色覚多様性

- 金額の増減・成功/失敗・警告は、色だけで伝えない。必ず記号、アイコン、文言、aria-label のいずれかを併用する。
  - 金額: `+` / `-` を併記する。
  - エラー/警告: 色 + アイコン + 文言にする。
  - グラフ: 色 + 軸ラベル + 目盛 + スクリーンリーダー向け要約を持たせる。
- 白文字を載せる塗り面は `--primary-fill` / `--primary-fill-strong` / `--success` / `--danger` を使う。
  `--primary` は淡い背景・フォーカス・枠など、文字を載せないアクセントに限定する。
- 赤/緑だけで意味を分けない。収支や残高は色に加えてラベルと符号で意味を固定する。
- 新しい色トークンを足す場合は、用途を `design-system.md` に追記し、最低でも本文/ボタン/状態表示の
  コントラストが WCAG AA 相当になることを確認する。
- E2E の axe-core は critical/serious 違反ゼロを守る。ただし axe は色覚シミュレーションを完全には代替しないため、
  UI 変更時は「グレースケールでも意味が読めるか」を目視確認する。

## 宗教・文化配慮

本アプリは個人の家計/生活コストを扱うため、価値観を押しつける文言やアイコンを避ける。

- 費目名・タグ名・管理区分名はユーザーが自由に変えられることを前提にする。初期値は日本語圏で自然な
  一般語に留め、特定宗教・思想・家族形態を前提にしない。
- 「娯楽」「交際」「健康」「経費」などのカテゴリは、善悪や浪費/節約の評価を含めない。意思決定用の
  分類であり、道徳評価ではない。
- アイコンは会計上の意味が明確な抽象記号を優先する。宗教的記号、国旗、特定文化の儀礼を連想させる
  記号を標準UIの意味表現に使わない。
- 日付/祝日/イベント名を標準機能に組み込む場合は、特定の暦や宗教行事を既定の正解として扱わない。
  必要ならユーザー定義タグやメモで扱う。
- 食費・医療・交際など、生活背景の差が出やすい項目に「おすすめ」「正常」「使いすぎ」などの断定的な
  判定文を出さない。表示は金額・推移・ユーザー自身の設定値に基づく。
